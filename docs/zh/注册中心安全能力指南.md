# 注册中心安全能力指南

注册中心安全能力如下：

- **安全通信**：系统间交互默认使用HTTPS协议，使用安全的TLS协议版本及算法套件。
- **访问控制**：提供认证回调函数，供定制实现。根据agent所有者进行AgentCard操作隔离，仅允许agent所有者修改、删除自己的AgentCard。
- **存储安全**：提供加解密回调函数，供定制实现，用于保护敏感数据；后台命令行的敏感入参使用交互式输入，避免敏感参数被系统日志记录。
- **日志审计**：默认记录到独立日志文件，记录关键操作的6个要素；同时提供日志审计回调函数，供定制实现。
- **AgentCard内容安全**：在注册阶段识别拦截恶意意图的AgentCard注册；默认校验agentcard完整性，同时提供注册中心的签名；提供agentcard人工审核能力。
- **证书生成工具**：提供独立工具生成自签名证书，用于调试场景。
- **验签公钥下载**：提供接口下载注册中心的验签公钥。

## 安全通信（系统间TLS通信）

注册中心提供AgentCard的注册、修改、删除等REST接口（参考[注册中心用户指南“接口能力”章节](./注册中心用户指南.md#接口能力)），这些接口默认使用HTTPS协议以保障通信通道的安全性，使用安全的TLS协议版本及算法套件<br>
- TLS协议版本：TLSv1.3、TLSv1.2<br>
- 算法套件：TLS_AES_256_GCM_SHA384,TLS_AES_128_GCM_SHA256,TLS_ECDHE_ECDSA_WITH_AES_256_GCM_SHA384,TLS_ECDHE_ECDSA_WITH_AES_128_GCM_SHA256,TLS_ECDHE_RSA_WITH_AES_256_GCM_SHA384,TLS_ECDHE_RSA_WITH_AES_128_GCM_SHA256,TLS_DHE_RSA_WITH_AES_256_GCM_SHA384,TLS_DHE_DSS_WITH_AES_256_GCM_SHA384,TLS_DHE_RSA_WITH_AES_128_GCM_SHA256,TLS_DHE_DSS_WITH_AES_128_GCM_SHA256<br>
> **说明**：协议版本及算法套件不支持修改<br>

注册中心的证书由用户提前准备好，证书要求：<br>
- 身份证书server.cer：
必选，仅支持pem编码格式<br>
证书格式：X.509v3<br>
证书密钥算法、密钥长度：RSA(>= 3072 bits)，ECDSA(>= 256 bits)<br>
有效期：当前时间有效<br>
证书密钥用法：签名、密钥加密<br>
扩展密钥用法：服务器身份验证<br>

- 私钥文件server_key.pem：
必选，仅支持pem编码格式<br>
私钥与公钥的匹配性：需要与server.cer中的公钥匹配<br>
该私钥文件需要由私钥口令保护，私钥口令需满足复杂度要求：至少8个字符，至少包含两种字符(数字、大写字母、小写字母、特殊字符`` `~!@#$%^&*()-_=+ | [{}]);:'",<.>/? ``和空格)<br>

调测场景可使用[自签名证书生成工具](#自签名证书生成工具)生成符合如上要求的两本证书文件，注意该证书不可用于生产环境。<br>

- 信任证书trust.cer：
默认必选，仅支持pem编码格式，仅支持.cer文件，如果涉及多本证书，需合成一本<br>
开启客户端证书校验场景下，该文件必须存在<br>
校验证书格式：X.509v3<br>
校验有效期：当前时间有效<br>
密钥算法、长度：RSA(>= 3072 bits)，ECDSA(>= 256 bits)<br>

- 吊销列表revocationlist.crl：
可选，仅支持pem编码格式，仅支持.crl文件，如果涉及多本证书，需合成一本，可以不存在<br>
校验证书格式：X.509v2<br>
校验有效期：当前时间有效<br>
不支持国密证书<br>

证书文件及所在目录的权限最小化(例如文件权限400/600，目录权限700)，同时需确保本项目进程拥有对文件的读取权限<br>

通过init命令将证书配置给注册中心，该命令中将配置相关证书路径，交互式输入私钥口令
```bash
# init 命令客户端启动并进入界面
[user@host registry-center-main]# python -m agent_registry.init
Enable HTTPS (y/n, default: true):  y

Configure server TLS certificate (RSA only):
# 支持指定相对路径和绝对路径。相对路径是基于项目根路径
Enter server certificate path ssl_certfile: (current: etc/ssl/server.cer): /testdir/test_server.cer
Enter server private key path ssl_keyfile: (current: etc/ssl/serve_key.pem): /testdir/test_key.pem
Enter server trust certificate path ssl_ca_certs: (current: etc/ssl/trust.cer): /testdir/trust.cer
# 吊销证书列表，可以不填
Enter server CRL file path ssl_cert_certs:
# ssl_keyfile文件更换默认位置才会要求输入密码
Enter server private key password:
# 密码不满足复杂度会有提示
Private key password complexity is low (Include at least two character types), continue using this password? (y/n):  y

# 开启客户端证书校验
Enable client certificate verification verify_client (y/n, default: false): y
```

本项目仅读取使用这些证书，不提供证书管理能力，例如证书过期告警、备份恢复等<br>
配置后重启生效<br>

注册中心同时提供http通信能力，可通过初始化配置命令行关闭https，启用http
```bash
# init 命令客户端启动并进入界面
[user@host registry-center-main]# python -m agent_registry.init
Enable HTTPS (y/n, default: true):  n
```
配置后重启生效<br>

## 访问控制

### 认证

提供认证回调函数，供按需定制实现，无默认的认证机制。定制实现方式参考[注册中心开发指南“自定义处理器使用说明”章节](./注册中心开发指南.md#自定义处理器使用说明)。

### AgentCard操作隔离

在注册中心开启了https通信且开启了客户端证书校验场景下，默认开启AgentCard操作隔离功能，支持关闭。<br>
注册中心以客户端TLS身份证书的CN字段作为操作者身份，以AgentCard注册REST接口的操作者作为agent所有者<br>
在AgentCard修改、删除REST接口中，会判断操作者是否为agent所有者，仅允许agent所有者对AgentCard进行修改、删除操作。<br>

未开启操作隔离的配置场景有：<br>
- http通信、或https通信但未开启客户端证书校验的场景；<br>
- 开启了https通信且开启了客户端证书校验，但操作隔离开关已关闭的场景（owner.isolation.enabled配置为false）。<br>
- 开启了https通信且开启了客户端证书校验，操作隔离开关已开启，校验模式为非严格校验模式（owner.validation.mode=relaxed），且AgentCard注册时操作者身份为空的场景。<br>
<br>
此场景下不做操作隔离，所有客户端均可执行任意AgentCard进行修改、删除操作。此机制仅适用于可信域内的交互，否则agentcard有被篡改的风险。<br>
```
# server.conf 配置文件

# 是否开启owner校验逻辑
owner.isolation.enabled=true

# Owner 校验模式
# strict：严格模式，CN不能为空。如果CN为空，则会拒绝注册请求
# relaxed：非严格模式，CN可以为空。如果CN为空，则不做操作隔离
owner.validation.mode=relaxed
```

## 存储安全

1. 提供加解密回调函数，供定制实现，用于保护敏感数据。定制实现方式参考[注册中心开发指南“自定义处理器使用说明”章节](./注册中心开发指南.md#自定义处理器使用说明)。

2. 后台命令行的敏感入参使用交互式输入，避免敏感参数被系统日志记录。例如自签名证书生成工具的私钥口令入参、init命令中的私钥口令入参与数据库密码入参。<br>

## 日志审计

针对关键操作和系统状态变更记录审计日志。可审计的关键操作有：<br>
- 注册中心初始化配置
- 注册中心服务启动
- 注册中心服务停止
- AgentCard注册
- AgentCard修改
- AgentCard删除
- AgentCard审核
- AgentCard标签设置
- 创建标签
- 修改标签
- 删除标签

记录关键操作的6个要素：时间、客户端IP、用户名称、操作名称、操作对象、操作结果。<br>

有两种记录方式：
1. 默认记录到独立日志文件<br>
文件路径：log/audit/audit.log<br>
文件大小上限5M，达到上限后，将存量日志文件从最大编号开始依次按编号+1重命名，例如audit.log.1 -> audit.log.2、audit.log -> audit.log.1<br>
数字编号超出4时，则老化删除该文件。数字编号越小，存的是越新的日志，最新的日志总是记录在audit.log中<br>
文件个数最多5个：audit.log、audit.log.1、audit.log.2、audit.log.3、audit.log.4<br>
文件大小、文件个数上限可配置，配置文件路径etc/conf/log_config.conf，配置项分别为
```properties
audit_log_max_file_size_mb=5
audit_log_backup_count=4
```

记录格式为json字符串，样例如下：<br>
- 服务启动操作：

## AgentCard内容安全

AgentCard仅支持仅支持中文、英文，AgentCard中不可携带敏感/机密数据及个人信息，否则存在信息泄露风险。<br>

### 拦截恶意意图的AgentCard注册

AgentCard注册REST接口中，会针对AgentCard中的如下字段做prompt注入校验、高危skill检查<br>
- description
- skill.name
- skill.description
- skill.tags

涉及如下黑名单的描述关键字，将被拒绝注册。此机制为默认机制，不可关闭<br>

> prompt注入关键词和高危skill描述关键词完整黑名单列表详见：[AgentCard安全注册规范 - 附录：完整黑名单列表](../../design/AgentCard_Security_Specification.md#7-附录完整黑名单列表)


### Agentcard完整性校验

根据Agentcard中的签名字段检查Agentcard是否被篡改。<br>
agentcard签名样例，如下signatures字段：<br>
```json
{
  "name": "Energy Saving Agent",
  "description": "负责能效优化的自主闭环运行，包括意图探索、意图实现、效果评估与报告。",
  ...
  "signatures": [
    {
      "protected": "eyJhbGciOiJFUzI1NiIsInR5cCI6IkpPU0UiLCJraWQiOiJrZXktMSIsImprdSI6Imh0dHBzOi8vZXhhbXBsZS5jb20vYWdlbnQvandrcy5qc29uIn0",
      "signature": "QFdkNLNszlGj3z3u0YQGt_T9LixY3qtdQpZmsTdDHDe3fXV9y9-B3m2-XgCpzuhiLt8E0tV6HXoZKHv4GtHgKQ"
    }
  ]
}
```
其中protected字段可解码为<br>
```json
{"alg":"ES256","typ":"JOSE","kid":"key-1","jku":"https://example.com/agent/jwks.json"}
```
注册中心将按照https://a2a-protocol.org/latest/specification/#84-agent-card-signing中的机制进行签名校验<br>
只要有一个签名校验通过，则认为完整性校验成功。<br>

签名算法支持RS256、ES256<br>
验签公钥可配置在后台文件：etc/sign_verify/jwks/{organization}/{agentname}.json<br>
文件内容是标准jwks格式，样例如下：<br>
```json
{
  "keys": [
    {
      "kty": "RSA",
      "use": "sig",
      "kid": "rsa-key-01",
      "alg": "RS256",
      "n": "0vx7agoebGcQSuu********JzKnqDKgw",
      "e": "AQAB"
    },
    {
      "kty": "EC",
      "use": "sig",
      "kid": "ec-key-01",
      "alg": "ES256",
      "crv": "P-256",
      "x": "MKBCTNIcKUS******KPAqv7D4",
      "y": "4Etl6SRW2Y******tmWWlbbM4IFyM"
    }
  ]
}
```

或携带在签名的protected字段jku中，注册中心将自动按GET方法调用该接口查询签名公钥进行验签。此方式获取到的公钥不缓存。<br>

该完整性校验功能默认启用，支持关闭。<br>
关闭方法：<br>
init命令<br>
```bash
# 启动注册中心init配置
[user@host registry-center-main]# python -m agent_registry.init
# 关闭AgentCard验签能力
Enable signature validation (y/n, default: true): n
Signature validation disabled
```


### 提供注册中心签名

agentcard注册、修改成功后，可为该agentcard计算注册中心签名。<br>
注册中心将按照https://a2a-protocol.org/latest/specification/#84-agent-card-signing中的机制使用注册中心的签名密钥完成签名。<br>
agent客户端从注册中心查询到的agentcard签名字段中，会同时包含原始agentcard签名及注册中心签名。<br>

在init命令中配置是否启用该功能，命令样例：<br>
Enable registry signing registry.sign.enabled (y/n, default: false): y
<br>

```bash
# init 命令客户端启动并进入界面
[user@host registry-center-main]# python -m agent_registry.init
# 其他配置项

# 开启注册中心签名
Enable registry signing registry.sign.enabled (y/n, default: false): y

Configure signing certificate (RSA only):
Enter signing certificate path sign_certfile:  /testdir/test_client.cer
# sign_keyfile文件更换默认位置才会要求输入密码
Enter signing private key path sign_keyfile: /testdir/test_client_key.pem
Enter signing private key password:
# 密码不满足复杂度会有提示，接收风险后可继续使用该低复杂度口令的私钥
Private key password complexity is low (At least 8 characters), continue using this password? (y/n):  y
```

该签名功能默认启用，启用时需配置注册中心签名证书。证书要求：<br>
- server.cer：
必选，身份证书，仅支持pem编码格式<br>
证书格式：X.509v3<br>
证书密钥算法、密钥长度：RSA(>= 3072 bits)，ECDSA(>= 256 bits)<br>
有效期：当前时间有效<br>
证书密钥用法：签名、密钥加密<br>
扩展密钥用法：服务器身份验证<br>

- server_key.pem：
必选，私钥文件，仅支持pem编码格式<br>
私钥与公钥的匹配性：需要与server.cer中的公钥匹配<br>
该私钥文件需要由私钥口令保护，私钥口令需满足复杂度要求：至少8个字符，至少包含两种字符(数字、大写字母、小写字母、特殊字符`` `~!@#$%^&*()-_=+ | [{}]);:'",<.>/? ``和空格<br>

验签公钥可通过接口GET /rest/v1/registry-center/keys获取，该接口无请求参数，会将init命令中配置的签名公钥证书sign_certfile中的公钥内容按标准jwks格式返回。接口定义可以参考[注册中心API参考](./注册中心API参考.md#获取公钥信息)<br>

调测场景可使用[自签名证书生成工具](#114-自签名证书生成工具)生成符合如上要求的两本证书文件，注意该证书不可用于生产环境。<br>


### Agentcard人工审核

注册中心提供人工审核能力，供管理员从法律与数据合规、业务逻辑与质量、资源及成本等维度审核已注册的Agent。<br>
默认不启用，启用方式：<br>
init命令行<br>
```bash
# 启动注册中心init配置
python -m agent_registry.init
# 配置是否开启审核开关
Enable agent approval (y/n, default: false): y
Approval function enabled
```
修改后重启生效，如果存在待审核的agent，则需将agent审核通过或删除后，才可关闭审核功能。<br>

Agent状态有registered、published。AgentCard查询、语义检索接口中，仅可查询到published状态的AgentCard。<br>
未启用该功能时，已注册的Agent默认状态为published。<br>
启用审核功能后，已注册的Agent默认状态为registered，管理员通过cli命令行可查询到全量agent信息（包括状态不为published的agent）。 <br>
管理员可继续通过cli命令行，查询态为registered的AgentCard，进行人工审核。<br>
管理员认可该agent后，可执行cli命令行审批该agent，该agent的状态将变为published。<br>
cli命令行使用方式请参考用户指南中[管理能力（CLI）](#121-管理能力cli)，包含Agent管理和标签管理等操作。

##  自签名证书生成工具

提供独立工具生成自签名证书，用于调测场景，注意该证书不可用于生产环境。<br>
工具命令行执行时，需指定生成证书的路径、私钥口令、证书用途。<br>
其中：<br>
- 证书路径支持输入相对路径，填写相对于 generate_selfsign_cert.py 所在目录的路径<br>
- 私钥口令为交互式输入<br>

证书用途支持两种：serverAuth、dataSigning，所生成的证书密钥用法、扩展用法不同：<br>
1. serverAuth：生成TLS通信证书<br>
证书密钥用法：数字签名、密钥加密<br>
扩展密钥用法：服务器身份验证<br>
命令行样例：<br>
```bash
# python generate_selfsign_cert.py {证书所在目录} {证书类型}
python generate_selfsign_cert.py testDir serverAuth
```

2. dataSigning：生成签名证书<br>
证书密钥用法：数字签名、不可否认<br>
命令行样例：
```bash
mkdir testDir
# python generate_selfsign_cert.py {证书所在目录} {证书类型}
python generate_selfsign_cert.py testDir dataSigning
Enter private key password:
Private key password complexity is low (At least 8 characters), continue using this password? (y/n):  y
Successfully generated self-signed certificates in testDir
```

执行成功后，会在指定路径下生成pem编码的X.509v3格式的自签名证书:<br>
证书密钥算法为RSA，密钥长度3072，证书有效期99年，签发者与使用者(issuer和subject的CN字段)均为agent-registry，不设置SAN。证书文件权限600，所在目录权限700。<br>
