# 注册中心用户指南

## 特性介绍

注册中心是一个专注于Agent统一管理的服务，支持用户将来自不同厂商的Agent进行集中注册与管理，实现多源Agent的可控接入与维护。

### 应用场景

应用于A2A-T领域多厂商多智能体交互场景下的AgentCard统一注册和管理。

### 能力范围

1. 本项目提供Agent注册中心模块供客户系统集成，用于管理客户系统内部的AgentCard，提供AgentCard注册、语义检索等能力，具备TLS通信、日志审计、访问控制等安全特性。

2. 本项目仅用作功能模块，非完整系统，模块自身不提供登录认证、鉴权、用户管理、加解密、密钥管理能力，需由客户系统提供如上安全基础设施；源码中已预留相关回调函数，供定制实现。

### 设计约束
1. 本项目需要运行在linux系统上，支持ipv4环境

2. 需要作为内部系统部署，不可开放到公网，不可作为云服务部署，否则目标系统需要同步提供防火墙、提供web服务器实现认证鉴权等安全能力。

3. 向本项目注册的AgentCard不可含有个人数据例如电话号码，不可含有敏感信息例如密码、凭据，否则有信息泄露风险。

4. AgentCard仅支持中文、英文两种语言。

## 接口能力

- **注册AgentCard**：支持将不同厂商的Agent注册到中心，统一纳管。详见[注册中心API参考](./注册中心API参考.md#注册agentcard)。
- **查询AgentCard列表**：根据指定条件查询符合条件的AgentCard列表。详见[注册中心API参考](./注册中心API参考.md#查询agentcard列表)。
- **查询指定AgentCard**：按AgentCard名称和组织精确查找唯一的AgentCard实例。详见[注册中心API参考](./注册中心API参考.md#查询指定agentcard)。
- **更新指定AgentCard**：更新指定AgentCard的信息。详见[注册中心API参考](./注册中心API参考.md#更新指定agentcard)。
- **删除指定AgentCard**：删除不再使用的AgentCard。详见[注册中心API参考](./注册中心API参考.md#删除指定agentcard)。
- **按语义检索AgentCard**：根据自然语言语义检索相匹配的AgentCard。详见[注册中心API参考](./注册中心API参考.md#按语义检索agentcard)。

## 配置文件速查

### 服务配置 (etc/conf/server.conf)

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| IP | 服务监听IP | 127.0.0.1 |
| PORT | 服务监听端口 | 5000 |
| enable_https | 启用HTTPS | true |
| ssl_certfile | 服务证书路径 | etc/ssl/server.cer |
| ssl_keyfile | 服务私钥路径 | etc/ssl/server_key.pem |
| signature_validation_enabled | 签名验证开关 | true |
| agent_approval_enabled | 审核开关 | false |
| use_vectordb | 启用向量数据库 | false |

### 持久化配置 (etc/conf/persistence.conf)

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| persistence.mode | 存储模式（file/postgresql） | file |
| postgresql.host | PostgreSQL主机 | 127.0.0.1 |
| postgresql.port | PostgreSQL端口 | 5432 |
| postgresql.name | 数据库名称 | registry_center |

### 高级配置 (etc/conf/server.properties)

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| agent.num.max | Agent数量上限 | 100 |
| connection.max | 最大连接数 | 500 |
| connection.timeout | 超时（秒） | 300 |

# FAQ

### Q1：注册Agent时提示"重复Agent"怎么办？

该Agent的(name, organization)组合已存在，需要：
1. 使用不同的名称或组织
2. 或使用更新接口修改现有Agent
3. 或先删除再注册

### Q2：签名验证失败怎么办？

可能原因：
1. 未配置验签公钥
2. AgentCard未包含signatures字段
3. 签名算法不匹配（仅支持RS256、ES256）
4. JKU URL无法访问

### Q3：如何关闭签名验证？

修改`etc/conf/server.conf`：
```properties
signature_validation_enabled=false
```

### Q4：如何配置PostgreSQL存储？

1. 修改`etc/conf/persistence.conf`
2. 创建数据库后重启服务

### Q5：Windows环境能否运行？

可以。注册中心支持Windows环境用于开发调试，启动入口统一为：`python -m agent_registry.start`。Windows下内置服务使用TCP协议（127.0.0.1:1108）。
